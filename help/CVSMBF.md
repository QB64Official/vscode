<blockquote>

### The CVSMBF function decodes a 4-byte STRING generated by MKSMBF$ (or read from a file) to SINGLE numeric values.

</blockquote>

#### SYNTAX

<blockquote>

`result! = CVSMBF ( stringData$ )`

</blockquote>

#### DESCRIPTION

<blockquote>


* CV functions ( [CVD](CVD.md) , [CVS](CVS.md) , [CVI](CVI.md) , [CVL](CVL.md) , [CVDMBF](CVDMBF.md) , [CVSMBF](CVSMBF.md) ) are used to convert values encoded by MK$ functions ( [MKD&dollar;](MKD&dollar;.md) , [MKS&dollar;](MKS&dollar;.md) , [MKI&dollar;](MKI&dollar;.md) , [MKL&dollar;](MKL&dollar;.md) , [MKDMBF&dollar;](MKDMBF&dollar;.md) , [MKSMBF&dollar;](MKSMBF&dollar;.md) ).
* QB64 has [_CV](CV.md) and [_MK&dollar;](MK&dollar;.md) functions which can also deal with extended data types .
* [SINGLE](SINGLE.md) values can range up to 7 decimal point digits. Decimal point accuracy depends on whole value places taken.

</blockquote>

#### EXAMPLES

<blockquote>



##### Example 1: Showcases the reduced space to store an encoded number.
```vb
a! = 700.2213
PRINT "Value of a!:"; a!
b$ = MKSMBF$(a!)
PRINT "Value of a! encoded using MKSMBF$: "; b$
PRINT "The string above, decoded using CVSMBF:"; CVSMBF(b$)
```
